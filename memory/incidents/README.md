# Post-Incident Notes

Detailed write-ups after significant issues. More depth than friction log.

## Format
Each incident gets its own file: `YYYY-MM-DD-brief-title.md`

```markdown
# Incident: [Title]

**Date:** YYYY-MM-DD
**Severity:** Low / Medium / High / Critical
**Duration:** How long was it broken?

## What Happened
[Narrative of what went wrong]

## Root Cause
[Why did this happen?]

## Resolution
[How was it fixed?]

## Lessons Learned
1. [Lesson 1]
2. [Lesson 2]

## Action Items
- [ ] Preventive measure 1
- [ ] Preventive measure 2
```
